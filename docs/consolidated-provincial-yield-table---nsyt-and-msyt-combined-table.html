The Consolidated Provincial Yield Table

Chapter 5 Consolidated Provincial Yield Table - NSYT and MSYT Combined Table

5.1 Overview

The following creates a single provincial yield table by integrating both NSYT generated through the VDYP workflow and MSYT generated through the provincial MSYT process. The objective is to produce a standardized yield dataset for every forested feature_id and opening in the provincial inventory, providing merchantable volume and dominant height projections across a common age range. The final product contains age-based estimates for conifer and deciduous merchantable volume (MVcon and MVdec) as well as conifer and deciduous dominant height (HTcon and HTdec), creating a unified yield layer that can be used in timber supply analysis, forest estate modelling, carbon assessments, and other initiatives.

The first component of the workflow processes the VDYP-generated natural stand yield projections. Species composition information is extracted from the VDYP yield tables and evaluated against a predefined list of deciduous species. A custom function separates projected merchantable volume into conifer and deciduous components by removing deciduous species volumes from the total projected volume. Age-specific projections are then converted from a long format into a wide yield table where each age step becomes a separate volume field. This process creates a complete set of conifer and deciduous merchantable volume projections for all natural stands represented by the VDYP yield system.

The second component processes the provincial MSYT managed stand yield database. Existing MSYT outputs are loaded together with reference tables identifying which features use managed stand yields. Because MSYT curves are typically generated to age 120, the script extends all volume and height projections to age 250 by carrying forward age-120 values for older ages. The resulting managed stand yield curves are standardized into the same structure as the VDYP outputs, ensuring that merchantable volume and dominant height fields are available at identical age intervals and can be combined seamlessly with natural stand projections.

To ensure consistency across both yield systems, the script also develops conifer and deciduous dominant height projections for VDYP stands. Using BEC-specific height ratio tables, dominant heights are separated into conifer and deciduous components based on the leading species and associated ecological region. The processed VDYP yields are then combined with the MSYT yields into a single provincial output table. Finally, quality assurance checks identify any records that contain zero merchantable volume across all age classes, allowing both NSYT and MSYT records to be reviewed for potential data issues before the final combined yield table is exported. The resulting dataset provides a comprehensive provincial yield library that supports a consistent yield curve for every feature_id in the corporate inventory and establishes a common foundation for provincial-scale forest analysis and modelling.

This script generates a combined yield table containing:

  • Conifer merchantable volume (MVcon_*)
  • Deciduous merchantable volume (MVdec_*)
  • Conifer dominant height (HTcon_*)
  • Deciduous dominant height (HTdec_*)

The workflow combines:

  1. VDYP-based yield curves (NSYT workflow)
  2. MSYT yield curves

The final output is exported as a single yield table for downstream modeling.

NoDecidVol_calculator() calculates conifer-only projected volume by removing deciduous species from VDYP projected species volumes.

NoDecidVol_calculator <- function (sp_list,
                                   vol_list,
                                   decidsp,
                                   multilayer = FALSE) {

  # Standardize species slot names
  sp_list[spcode %in% "SPECIES_1_CODE", `:=`(spcode, "sp1")]
  sp_list[spcode %in% "SPECIES_2_CODE", `:=`(spcode, "sp2")]
  sp_list[spcode %in% "SPECIES_3_CODE", `:=`(spcode, "sp3")]
  sp_list[spcode %in% "SPECIES_4_CODE", `:=`(spcode, "sp4")]
  sp_list[spcode %in% "SPECIES_5_CODE", `:=`(spcode, "sp5")]
  sp_list[spcode %in% "SPECIES_6_CODE", `:=`(spcode, "sp6")]

  # Remove deciduous species
  sp_list2 <- sp_list[
    !is.na(sp) &
      sp != "" &
      !sp %in% decidsp
  ]

  # Replace NA projected volumes with 0
  vol_list[is.na(PRJ_SP1_VOL_DWB), `:=`(PRJ_SP1_VOL_DWB, 0)]
  vol_list[is.na(PRJ_SP2_VOL_DWB), `:=`(PRJ_SP2_VOL_DWB, 0)]
  vol_list[is.na(PRJ_SP3_VOL_DWB), `:=`(PRJ_SP3_VOL_DWB, 0)]
  vol_list[is.na(PRJ_SP4_VOL_DWB), `:=`(PRJ_SP4_VOL_DWB, 0)]
  vol_list[is.na(PRJ_SP5_VOL_DWB), `:=`(PRJ_SP5_VOL_DWB, 0)]
  vol_list[is.na(PRJ_SP6_VOL_DWB), `:=`(PRJ_SP6_VOL_DWB, 0)]

  # Convert projected species volumes to long format
    vol_list2 <- melt(
      vol_list,
      id.vars = c("FEATURE_ID", "PRJ_TOTAL_AGE"),
      measure.vars = c(
        "PRJ_SP1_VOL_DWB",
        "PRJ_SP2_VOL_DWB",
        "PRJ_SP3_VOL_DWB",
        "PRJ_SP4_VOL_DWB",
        "PRJ_SP5_VOL_DWB",
        "PRJ_SP6_VOL_DWB"
      ),
      variable.name = "spcode",
      value.name = "vol"
    )

  # Standardize projected volume species names
  vol_list2[spcode %in% "PRJ_SP1_VOL_DWB", `:=`(spcode, "sp1")]
  vol_list2[spcode %in% "PRJ_SP2_VOL_DWB", `:=`(spcode, "sp2")]
  vol_list2[spcode %in% "PRJ_SP3_VOL_DWB", `:=`(spcode, "sp3")]
  vol_list2[spcode %in% "PRJ_SP4_VOL_DWB", `:=`(spcode, "sp4")]
  vol_list2[spcode %in% "PRJ_SP5_VOL_DWB", `:=`(spcode, "sp5")]
  vol_list2[spcode %in% "PRJ_SP6_VOL_DWB", `:=`(spcode, "sp6")]

  # Merge species and volume tables
    all <- merge(
      sp_list2,
      vol_list2,
      by = c("FEATURE_ID", "spcode"),
      all.x = TRUE
    )

    all2 <- all[
      ,
      .(PRJ_VOL_DWB_conifer = sum(vol, na.rm = TRUE)),
      by = c("FEATURE_ID", "PRJ_TOTAL_AGE")
    ]

  return(all2)
}

5.3.3 Separate conifer height and deciduous height

5.3.3.1 Height ratios

The following develops a set of BEC-specific conifer-to-deciduous dominant height ratios that are subsequently used to separate VDYP projected dominant heights into conifer and deciduous components for the provincial yield table. The workflow begins by connecting to the provincial MSYT PostgreSQL database and loading the provincial managed stand yield dataset. From the MSYT outputs, only features containing valid conifer (HTcon_120) and deciduous (HTdec_120) dominant heights at age 120 are retained. These features are then joined to their corresponding Biogeoclimatic Ecosystem Classification (BEC) zone and subzone information, creating a unique BEC label for each feature. For every stand, the script calculates both the deciduous-to-conifer height ratio (HTdec_120 / HTcon_120) and the conifer-to-deciduous height ratio (HTcon_120 / HTdec_120). These ratios are then summarized by BEC zone and subzone, producing average height relationships for each ecological unit based on all available managed stand observations. The resulting table contains the number of contributing samples and mean height ratios for each BEC variant and is exported as bec_height_ratios.csv. These ratios are subsequently used within the provincial NSYT workflow to estimate separate conifer and deciduous dominant height projections from a single VDYP dominant height prediction, ensuring consistency between the provincial managed stand and natural stand yield datasets.

  mutate(

    HTcon = case_when(
      is_decid ~ PRJ_DOM_HT * con_dec_ratio,
      TRUE ~ PRJ_DOM_HT

      is_decid ~ PRJ_DOM_HT,
      TRUE ~ PRJ_DOM_HT * dec_con_ratio

  )
